# Niuva Brand Website Approval Decisions

- Phase: 0.2 - Brand Website Approval Decisions
- Status: Approved for Homepage prototype work
- Scope: Public Homepage concepts only
- Decision date: 13 July 2026
- Audit reference: `doc/brand/BRAND_WEBSITE_AUDIT.md`

## 1. Purpose

This document records the decisions approved after the Phase 0.1 Niuva Brand-to-Website Audit. It defines what the Phase 1 Homepage prototypes may explore, what must remain identical between concepts, what foundations are protected, and what decisions remain deferred.

This approval does not authorize a global redesign, shared design-system migration, or rollout to any route other than isolated Homepage concept prototypes.

## 2. Decision status vocabulary

| Status | Meaning |
|---|---|
| Approved | The decision may guide Phase 1 prototype work |
| Approved with constraints | The decision may be explored only within the listed limits |
| Protected | The foundation must remain unchanged during prototype work |
| Deferred | No implementation decision may be made until prototype review |
| Not authorized | The work is outside the current scope |

## 3. Approved visual direction

### Decision

**Status: Approved with constraints**

The approved preferred direction is:

**Concept B: Experimental Engineering Studio**

Concept B must apply the editorial clarity, readability, and restraint established by Concept A. Experimental expression must come from meaningful composition, project evidence, graphic behavior, and controlled motion. It must not come from decorative technology effects.

### Required personality

The Homepage concepts should communicate:

- Experimental
- Technical
- Collaborative
- Approachable
- Precise
- Forward-looking

### Rejected perception

The Homepage concepts must not feel:

- Overly corporate
- Like a generic SaaS website
- Futuristic only for decoration
- Card-heavy or bento-heavy
- Like a direct recreation of the Brand Guidelines PDF
- Like an enterprise consulting catalogue
- Like a static company profile transferred into HTML

### Direction boundary

Concept B is the preferred direction, but Concept A remains required as the control concept. The purpose of Phase 1 is to compare both concepts using identical factual content and behavior before approving a final Homepage composition and public typography pair.

## 4. Typography prototype decision

### Decision

**Status: Approved for comparison, final selection deferred**

No final public font family is approved in Phase 0.2.

Phase 1 must prototype and compare the following typography systems.

### Concept A typography control

| Role | Typeface |
|---|---|
| Display | Poppins |
| Headings | Poppins |
| UI | Poppins |
| Body | Inter |
| Forms | Inter |
| Labels | Inter |
| Captions | Inter |

This pair is the Brand Guidelines v1.0 control. It tests how much studio personality can be created through composition, imagery, graphic behavior, and motion without changing the approved brand typefaces.

### Concept B preferred typography experiment

| Role | Typeface |
|---|---|
| Display | Manrope |
| Headings | Manrope |
| UI | Manrope |
| Body | Manrope |
| Short technical annotations | Space Mono |

This pair is approved only as a Homepage prototype experiment. It is not approved for production rollout.

### Typography rules

1. JetBrains Mono is not part of the future public visual direction.
2. Existing JetBrains Mono usage outside the isolated prototypes must not be changed during Phase 1.
3. Space Mono may be used only for short, factual technical annotations.
4. Space Mono must not be used for paragraphs.
5. Space Mono must not be used for buttons.
6. Space Mono must not be used for navigation.
7. Space Mono must not be used as a decorative shorthand for engineering.
8. Technical annotation must be attached to real information, such as a process state, project artifact, material, measurement, or output.
9. Both concepts must be tested with the same Indonesian headings and body content.
10. Mobile wrapping, readability, loading behavior, and layout stability must be evaluated before font approval.

### Final approval gate

The final font decision will be made after side-by-side Homepage prototype testing.

Any permanent move away from Poppins + Inter requires:

- Stakeholder approval
- A documented comparison result
- A Brand Guidelines v1.1 update
- Updated digital typography roles
- Responsive and performance verification

## 6. Approved graphic behavior

### U-curve decision

**Status: Approved with constraints**

The U-curve is approved as a transformation path representing:

**Need -> Research -> Experiment -> Prototype -> Output**

The U-curve should help visitors understand movement from an initial need toward a testable or usable result. It should function as narrative structure, not repeated decorative wallpaper.

### Permitted U-curve roles

- Connect the stages of a real product-development process.
- Guide the eye between a need, an artifact, and an output.
- Crop into or frame a real project image.
- Support one controlled transition between major Homepage chapters.
- Provide a static process path in reduced-motion mode.

### Prohibited U-curve roles

- Repeated background decoration in every section.
- A generic stamp added wherever space feels empty.
- A motif layered over text in a way that reduces readability.
- Multiple competing paths on the same viewport.
- A purely animated flourish with no informational role.

### Approved Home motif budget

| Element | Maximum |
|---|---:|
| Dominant motif placements | 2 |
| Secondary accent placements | 1 |
| Decorative motif in every section | 0 |

### Circle behavior

Circles may be used only as:

- Process nodes
- Start points
- Endpoints
- Decision points
- Focus markers tied to real content

Circles must not be repeated as arbitrary status dots, list bullets, or card decoration.

### Dot and line patterns

Dot or line patterns are permitted only when connected to a real technical context, such as:

- Measurement
- Iteration
- Testing
- Material behavior
- Component mapping
- Project documentation

Decorative grid backgrounds and repeated line textures are not approved.

| Asset | Approved prototype role | Restriction |

### Photography rules

- Do not create fake project photography.
- Do not imply that generated or stock imagery documents a real Niuva project.
- Do not use stock corporate meeting photography.

- Sketching and CAD work

## 9. Motion decision

### Decision

**Status: Moderate public motion approved with constraints**

Target motion intensity:

**5/10**

Motion must explain hierarchy, process, feedback, or state change. Motion added only for spectacle is not approved.

### Permitted motion budget

| Motion event | Maximum |
|---|---:|
| Hero entrance | 1 |
| U-curve process transformation | 1 |
| Project media transition | 1 |

### Permitted behavior

- A concise hero entrance that establishes hierarchy.
- One transformation showing Need to Output through the U-curve path.
- One transition that improves project-media comprehension.
- Transform and opacity-based movement where possible.
- Static visible content before motion enhancement.
- Motion cleanup and lifecycle safety.

### Prohibited behavior

- Perpetual motion.
- Animation on every section.
- Decorative parallax.
- Hover gimmicks.
- Cursor-following effects.
- Motion required to understand content.
- Scroll hijacking without a clear informational need.
- Multiple marquees.
- Infinite loops.
- Layout-property animation that risks jank.
- Motion that delays access to CTA or project evidence.

### Reduced-motion contract

`prefers-reduced-motion: reduce` must provide a complete static equivalent.

In reduced-motion mode:

- All content remains visible.
- The full Need to Output sequence remains understandable.
- Project media remains available without transition.
- CTA behavior remains unchanged.
- No information depends on timing, scrubbing, or animation state.

### Decision

**Status: Deferred until prototype review**

The following are not approved in Phase 0.2:

1. Final public font pair.
2. Final Homepage composition.
3. Permanent shared component API changes.
4. Permanent PageHero or SectionHeader migration.
5. Service taxonomy renaming.
6. Office & Signage placement.
7. Rollout to About.
8. Rollout to Capabilities.
9. Rollout to Projects.
10. Rollout to Contact.
11. Changes to admin, auth, dashboard, or order interfaces.
12. Brand Guidelines v1.1 updates.
13. Global font-token changes.
14. Global graphic-element rules.
15. Production motion rollout.

### Prototype review criteria

The review must compare both concepts against:

- Brand personality fit
- B2B credibility
- Creative-studio distinctiveness
- Content readability
- Mobile heading wrapping
- Project-evidence prominence
- Capability hierarchy clarity
- CTA visibility
- Photography quality and dependency
- Motif restraint
- Motion clarity
- Reduced-motion completeness
- Accessibility
- Performance risk
- Maintainability

No deferred decision becomes approved by appearing in a prototype. Approval must be recorded explicitly after the comparison review.
